 12.6改动思路：
 NIO+线程池的方式
 	主线程负责接收连接和请求，并阻塞，由线程池中的线程进行业务逻辑处理
 中继器的作用：
  	中继器只负责最开始的连接
 	中继器负责维护被控端的信息，控制端和被控端首次都先连接中继器，而后被控端作为服务器，控制端作为客户端进行点对点连接(nio+线程池)，且都断开和中继器的连接，
 	一旦控制端断开以后，被控端重新连接上中继器
